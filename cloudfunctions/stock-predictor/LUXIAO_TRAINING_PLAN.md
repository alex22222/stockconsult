# 露笑科技 (002617) 预测模型演进训练计划

> **目标**: 通过六阶段渐进式训练，使模型准确率从随机基线(50%)逐步提升至可交易水平(>55%)，并建立科学评估体系验证模型的真实预测能力。

---

## 一、股票画像与预测难点

### 1.1 基本信息
| 项目 | 内容 |
|------|------|
| 股票代码 | 002617 (深交所中小板) |
| 主营业务 | 碳化硅衬底材料、光伏发电、电磁线 |
| 行业分类 | 半导体材料(第三代) / 光伏设备 / 电力设备 |
| 市值特征 | 中小盘，高弹性 |

### 1.2 预测难点分析

| 难点类型 | 具体表现 | 对模型的挑战 |
|----------|----------|-------------|
| **高波动性** | 日波动率显著高于大盘 | 信号噪声比低，模型容易过拟合 |

| **技术驱动** | 碳化硅良率、产能爬坡等基本面变化 | 传统技术指标可能失效 |

## 二、六阶段训练计划

### Phase 1: 基础模型冷启动 (第1-7天)

**目标**: 建立基线模型，验证特征有效性

**步骤**:
1. 获取500日历史数据 (约2年)
2. 构建六大类因子特征 (市场环境、大盘能量、市场情绪、技术指标、板块热度、资金异动)
3. 5折时间序列交叉验证
4. 训练4模型集成 (GBDT、RandomForest、ExtraTrees、LogisticRegression)
5. 输出基线AUC和准确率

**成功标准**:
```
✓ CV AUC > 0.52 (显著高于随机)
✓ 所有4个模型训练成功
✓ 特征重要性分布合理 (技术指标占比最高)
✓ 无数据泄露迹象 (如未来函数)
```

**关键检查点**:
- [ ] 特征矩阵无NaN/Inf
- [ ] 目标变量构建正确 (明日涨跌，非当日)
- [ ] 时间顺序严格保持

---

### Phase 2: 滚动回测验证 (第8-21天)

**目标**: Walk-forward验证模型在不同时期的稳定性

**方法**:
```
训练窗口: 120日
预测窗口: 20日  
步长: 20日
覆盖: 约2年数据 → 15-18个滚动窗口
```

**成功标准**:
```
✓ 滚动AUC均值 > 0.53
✓ AUC标准差 < 0.08 (稳定性)
✓ 无持续下降趋势 (最新窗口AUC > 最早窗口AUC - 0.05)
✓ 上涨/下跌预测准确率差异 < 15% (不偏袒某一方向)
```

**风险信号**:
- 滚动AUC波动过大 → 模型不稳定，需增加正则化
- 近期窗口AUC显著下降 → 市场环境变化，需调整特征

---

### Phase 3: 外部环境因子注入 (第22-35天)

**目标**: 针对露笑科技特点，引入外部关联数据提升预测能力

**新增因子类别**:

| 类别 | 具体因子 | 数据源 |
|------|----------|--------|
| **关联板块指数** | 中证光伏产业指数收益率、中证半导体指数收益率、新能源车指数收益率 | AKShare指数数据 |
| **上下游关联个股** | 北方稀土/五矿稀土(上游)、比亚迪/宁德时代(下游)的1/5/20日收益率 | AKShare个股数据 |
| **行业资金流向** | 半导体板块净流入、光伏板块净流入 | AKShare板块资金流 |
| **宏观事件标记** | 半导体政策发布日、光伏补贴调整日、季报发布日 | 手动标注 + 新闻数据 |

**成功标准**:
```
✓ 加入外部因子后 AUC > 0.54
✓ 外部因子在特征重要性中占比 > 10%
✓ 无多重共线性问题 (VIF < 10)
```

---

### Phase 4: 在线学习与滚动训练 (第36-65天)

**目标**: 模拟每日在线学习，积累验证数据

**每日流程**:
```
08:00 获取昨日收盘数据
08:30 用最近120日数据重新训练模型
09:00 生成今日预测 (涨/跌 + 置信度)
15:00 收盘后记录今日实际涨跌
09:00(+1) 将昨日预测与实际对比，记录准确率
```

**成功标准**:
```
✓ 近20日准确率 > 52%
✓ 置信度与实际准确率正相关 (高置信度预测更准确)
✓ 模型无显著衰减信号
```

**衰减信号监控**:
| 信号 | 阈值 | 应对措施 |
|------|------|----------|
| 连续10日准确率 < 45% | 触发 | 扩大训练窗口、增加正则化 |
| AUC较上月下降 > 0.05 | 触发 | 重新特征工程、加入新因子 |
| 夏普比率转负 | 触发 | 暂停交易信号、仅保留预测 |

---

### Phase 5: 进化优化与超参搜索 (第66-86天)

**目标**: 基于积累的数据触发进化，优化模型

**进化策略**:
1. **因子权重进化**: 根据各类因子近期表现动态调整权重
2. **模型权重进化**: 根据各模型AUC表现用Softmax调整集成权重
3. **特征选择进化**: 每类保留Top-K，淘汰低贡献特征
4. **超参微调**: 对学习率、树深度等进行小范围搜索

**进化触发条件**:
```python
if 近期准确率 < 0.52 or 验证样本数 >= 30且间隔 >= 5天:
    trigger_evolution()
```

**成功标准**:
```
✓ 进化后AUC > 0.55
✓ 回测夏普比率 > 0.5
✓ 因子权重分布合理 (不过度依赖单一因子)
✓ 进化前后指标有提升 (AUC提升 > 0.01)
```

---

### Phase 6: 实盘模拟与持续监控 (第87天起，持续)

**目标**: 完整模拟实盘运行，持续监控模型健康度

**每日Pipeline**:
```
1. fetch_data()      → 获取最新数据
2. build_features()  → 构建特征 (含外部因子)
3. train_models()    → 滚动窗口训练
4. predict()         → 生成预测 + 置信度
5. record()          → 记录预测
6. next_day_verify() → 次日验证并更新进化状态
7. evolve_if_needed()→ 检查触发条件，执行进化
8. health_check()    → 运行合理性评估
```

**成功标准**:
```
✓ 近60日准确率 > 55%
✓ 最大回撤 < 15%
✓ 模型合理性评分 > 60/100
✓ 无严重模型衰减信号
```

## 三、科学评估体系

### 3.1 统计显著性检验

#### 二项检验
```
H0: 准确率 = 0.5 (随机猜测)
H1: 准确率 > 0.5 (有预测能力)

要求: p < 0.05 (即200次预测中至少需正确113次)
```

#### 置换检验
```
原理: 打乱标签后重新计算AUC，看真实AUC是否在置换分布的top 5%

要求: 置换p值 < 0.05
```

#### 游程检验
```
检测: 预测错误是否随机分布
若错误集中出现 → 模型在某些市场环境下系统性失效

要求: 不显著 (p > 0.05)，即错误分布随机
```

### 3.2 基准对比

| 基准策略 | 说明 | 击败标准 |
|----------|------|----------|
| 随机策略 | 随机做多/空仓 | 策略收益 > 随机均值 + 2σ |
| 买入持有 | 一直持有露笑科技 | 策略收益 > 买入持有 |
| 简单均线 | 5日均线上穿20日均线做多 | 策略收益 > 均线策略 |
| 反向策略 | 与模型预测反向操作 | 策略收益 > 反向策略 (否则模型方向反了) |

### 3.3 置信度校准

```
理想状态:
  预测70%上涨 → 实际应有70%上涨
  预测80%上涨 → 实际应有80%上涨

评估指标:
  ECE (Expected Calibration Error) < 0.1
  Brier Score 越低越好

校准差的后果:
  高置信度预测实际不准确 → 仓位管理失效
```

### 3.4 过拟合检测

```
警戒线:
  样本内AUC - 样本外AUC > 0.05 → 轻度过拟合
  样本内AUC - 样本外AUC > 0.10 → 严重过拟合

防范措施:
  1. 严格时间序列分割 (禁止使用未来数据)
  2. 滚动训练而非全量训练
  3. 特征数/样本数 < 1:10
  4. 使用简单模型作为正则化
```

### 3.5 综合评分卡

| 评估维度 | 权重 | 通过标准 | 评分方法 |
|----------|------|----------|----------|
| 统计显著性 | 25% | 二项检验p<0.05 & 置换检验p<0.05 | 通过得100，不通过得0 |
| 基准对比 | 20% | 击败随机 & 不输给反向 | 全部通过得100 |
| 置信度校准 | 15% | ECE < 0.1 | ECE<0.05得100, <0.1得60, 否则0 |
| 过拟合控制 | 15% | 内外差距 < 0.05 | 差距<0.03得100, <0.05得60 |
| 经济意义 | 15% | 扣除成本后净收益>0 & 信息比率>0.3 | 满足得100 |
| 稳定性 | 10% | CV<0.15 & 无下降趋势 | 稳定得100，不稳定得0 |

**最终裁决**:
- ≥80分: 模型通过合理性检验，具备预测能力
- 60-79分: 基本合理，有改进空间
- 40-59分: 合理性存疑，需深入分析
- <40分: 未通过检验，可能为过拟合

## 六、风险与应对

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|----------|
| 市场结构突变 | 中 | 高 | 扩大训练窗口、增加宏观因子 |
| 数据质量下降 | 低 | 中 | 多源校验、异常值检测 |
| 模型过拟合 | 中 | 高 | 正则化、简化模型、滚动训练 |
| 计算资源不足 | 低 | 低 | 简化特征集、降低训练频率 |
| 伪相关陷阱 | 中 | 高 | 严格统计检验、经济意义验证 |
