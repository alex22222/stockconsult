## 一、系统概述

### 模型架构

| 组件 | 说明 |
|------|------|
| **本地模型（sklearn 集成）** | GradientBoosting + RandomForest + ExtraTrees + LogisticRegression + AdaBoost + SVM + MLP，加权投票 |
| **云模型（五因子评分）** | 趋势、动量、量能、技术（RSI）、隔夜美股，线性加权 |
| **LSTM 深度学习** | LSTM + Attention，仅用于中国平安（601318）的单股实验 |
| **特征工程** | 100+ 特征，覆盖市场环境、大盘能量、市场情绪、技术指标、板块热度、资金异动、北向资金、涨跌停统计、国债收益率、隔夜美股、估值因子 |
| **回测引擎** | 滚动窗口训练，每 20 天再训练，含佣金/印花税/滑点 |

| **模拟盘** | 5 日持有期策略，信号生成 + 结算 + 周报 |
| **评估体系** | 二项检验、置换检验、游程检验、置信度校准（ECE/Brier）、过拟合检测、经济意义评估、稳定性分析 |

## 二、有帮助的地方（框架层面）

### 2.1 特征工程方向正确

六类因子的设计覆盖了 A 股短期走势的主要已知影响因素：

- 市场环境因子（指数趋势、波动率、个股与指数相关性）
- 大盘能量因子（成交量比率、价格动量、换手率变化）
- 市场情绪因子（恐惧贪婪指数、连续涨跌天数、振幅）
- 技术指标因子（MACD、KDJ、RSI、布林带、均线系统、OBV、ATR）
- 板块热度因子（相对强弱、板块资金流向）
- 资金异动因子（成交量异常、价量背离、主力资金）
- 宏观因子（北向资金、涨跌停、国债收益率、隔夜美股）

这些确实是学术界和业界公认对 A 股有预测力的因子类别。

### 2.2 回测方法论基本正确

- **滚动窗口训练**：每 retrain_days 天用过去 lookback_days 天数据重新训练
- **时间序列分割**：训练集在前，测试集在后，严格避免数据泄露
- **标准化隔离**：用训练集 fit scaler，测试集仅 transform
- **交易成本建模**：佣金 0.025%、印花税 0.1%（仅卖出）、滑点 0.1%、最低佣金 5 元
- **纯多头策略**：A 股 T+0 无法做空，选择「预测涨→持仓，预测跌→空仓」是合理方向

### 2.3 评估框架设计思路好

`model_evaluation.py` 中的检验体系是业界标准做法：

- 统计显著性：二项检验验证准确率是否显著高于 50%
- 置信度校准：ECE（Expected Calibration Error）和 Brier 分数
- 过拟合检测：样本内 vs 样本外指标差距
- 基准对比：随机策略、买入持有、动量策略
- 经济意义：扣除交易成本后的净收益、信息比率

### 2.4 集成策略优于单模型

多模型加权投票能降低单一模型的方差，evolution_engine 的动态权重调整也是合理设计。

### 第二层：日频预测可能不具备可交易性

#### 3.2.1 T+1 时间错配

A 股的结算规则与模型的预测目标存在根本性错配：

```
T 日收盘  →  模型生成信号（预测 T+1 涨跌）
T+1 开盘  →  根据信号买入（但 T+1 买入当天不能卖出）
T+1 收盘  →  模型判断预测是否正确（但你卖不掉）
T+2 开盘  →  实际可以卖出
```

模型预测的是 **T+1 收盘价 vs T 日收盘价**，但真实盈亏取决于 **T+2 开盘价 vs T+1 开盘价**。这是两个不同的随机变量。

回测中使用的是理想化的「当日收盘价买入/卖出」，在实际的 A 股 T+1 制度下根本无法执行。

#### 3.2.2 交易成本侵蚀

一笔完整的买入+卖出交易：

| 费用项 | 费率 | 10,000 元账户 |
|--------|------|:------------:|
| 买入佣金 | 0.025%（最低 5 元） | ~5 元 |
| 买入滑点 | 0.1% | ~10 元 |
| 卖出佣金 | 0.025%（最低 5 元） | ~5 元 |
| 卖出滑点 | 0.1% | ~10 元 |
| 印花税（仅卖出） | 0.1% | ~10 元 |
| **合计** | **~0.35%** | **~35-40 元** |

如果日均期望收益只有 0.1%，交易成本会吃掉三天半的利润。小资金账户（回测默认 10,000 元）的最低佣金影响尤其严重——频繁交易注定亏损。

#### 3.2.3 胜率 > 50% ≠ 赚钱

二项检验只验证了「不是纯随机」，但：
- 55% 胜率 + 盈亏比 1:1 → 扣除 0.35% 成本 → 期望收益几乎为零
- 需要胜率 > 60% 或盈亏比 > 1.5 才能在成本后盈利

模型中未见盈亏比分析。

---

### 第三层：风控体系严重不足

#### 3.3.1 无仓位管理

回测中只有两种状态：80% 资金满仓 或 空仓。缺少：

- **分级仓位**：信号强 → 重仓（60-80%），信号弱 → 轻仓（20-30%）
- **凯利公式**：根据胜率和盈亏比计算最优仓位比例
- **波动率倒数**：高波动时降低仓位
- **最大回撤熔断**：连续亏损 N 次后暂停交易

#### 3.3.2 无市场状态识别

模型对所有市场时期（牛市/熊市/震荡市）一视同仁。实际上：

- **牛市中**：频繁空仓会踏空，预测准确率虚高（因为多数天都在涨）
- **熊市中**：频繁交易会被趋势碾压，止损不断触发
- **震荡市中**：信号频繁翻转，交易成本累积

没有市场状态（regime）分类器来调节交易频率和仓位。

#### 3.3.3 无组合层面风控

目前是单只股票独立回测，缺少：

- 行业集中度限制
- 个股相关性矩阵
- 组合层面最大回撤限制
- 每日最大亏损熔断

---

### 第四层：A 股特殊规则未处理

| 规则 | 影响 | 是否处理 |
|------|------|:--:|
| T+1 结算 | 买入当天不能卖出 | ❌ |
| 涨跌停 ±10%/±20% | 涨停买不到，跌停卖不掉 | ❌ |
| 停牌 | 持仓无法卖出 | ❌ |
| 除权除息 | 价格跳空非真实涨跌 | ❌ |
| 熔断机制 | 极端行情暂停交易 | ❌ |

如果模型在涨停日发出买入信号，你实际上买不到——但回测中会按收盘价「成交」。

---

### 第五层：过拟合风险

#### 3.5.1 特征/样本比严重失衡

- 特征数量：**100+**
- 训练窗口：252 天
- 滚动特征需要 60 天预热 → 有效样本约 **190 天**
- 特征/样本比 ≈ **1:1.9**（理想情况应 > 1:10）

这是教科书级的过拟合风险条件。100 个特征在 190 个样本上做集成学习，模型几乎一定能找到「伪模式」。

#### 3.5.2 幸存者偏差

训练数据中的股票都是「至今仍在交易」的——已经退市的被排除在外。用「活下来的公司」训练出的模型，在真实市场中必然高估表现。

---

### 第六层：5 日策略与日频模型混用

系统内存在两套独立的预测目标：

| 组件 | 预测目标 | 持有期 |
|------|----------|:--:|
| `model_trainer.py` | 次日涨跌 | 1 天 |
| `backtest_engine.py` | 次日涨跌 | 1 天 |
| `paper_trading_5day.py` | 5 日累计收益 > 0 | 5 天 |
| `backtest_weekly.py` | 5 日累计收益 > 0 | 5 天 |
| `lstm_601318.py` | 次日涨跌 | 1 天 |

两套标签（1 日 vs 5 日）同时存在，没有统一的目标函数。实盘中到底按哪个执行？

---

### 第七层：数据与工程

- **离线 CSV，非实时数据流**：数据靠 `local_data_provider.py` 从本地 CSV 读取。实盘需要每个交易日 15:00 后自动拉取、更新特征、生成信号
- **无定时任务**：预测生成需要人工执行 Python 脚本
- **云函数未接入**：`cloudfunctions/stock-predictor/` 下面代码完善，但没有证据表明它在实际触发运行
- **前端未展示真实回测结果**：预测页面显示了「本地模型」和「云模型」的预测，但没有展示历史准确率或回测统计

## 四、优先级路线图

要让这个系统达到「可辅助交易决策」的水平，必须完成以下最小必要集：

### Phase 1：证明预测能力（先不做任何改进）

1. **跑完整回测，输出结果表**
   - 4 只股票分别在回测引擎中的：总收益、年化收益、夏普比率、最大回撤、胜率、交易次数
   - 对比基准：买入持有、简单均线策略、随机策略
   - 输出到文档，公开可见

2. **启动每日预测跟踪**
   - 每个交易日收盘后运行预测，记录到 JSON
   - T+1 收盘后自动验证
   - 至少积累 30 个交易日后再下结论

3. **运行 model_evaluation 综合评估**
   - 在真实股票数据上运行全部 6 项检验
   - 输出综合评分（0-100）

### Phase 2：修复基础问题

4. **修复 T+1 时间错配**
   - 将预测标签从「T+1 收盘 vs T 收盘」改为「T+2 开盘 vs T+1 开盘」
   - 或改用周度信号（持有 5 天），与 paper_trading_5day 统一

5. **加入仓位管理**
   - 信号置信度高 → 高仓位（60-80%）
   - 信号置信度低 → 低仓位（20-30%）或不交易
   - 连续亏损 N 次 → 暂停 M 天

6. **处理涨跌停**
   - 涨停日：买入信号自动跳过
   - 跌停日：卖出信号自动跳过
   - 在回测中标记不可执行信号

### Phase 3：降低风险

7. **特征降维**
   - 用 feature importance 筛选 Top 30 特征
   - 或使用 PCA / 正则化

8. **加入市场状态识别**
   - 简单的牛市/熊市/震荡分类器（基于指数 60 日均线斜率）
   - 不同状态下使用不同仓位或不同的置信度阈值

9. **接入实时数据**
   - akshare 定时拉取 → 自动生成信号
   - 或使用 tushare pro

### Phase 4：进阶（Phase 1-3 确认有正向预期收益后）

10. 组合层面风控（多只股票协方差、行业集中度）
11. 实盘模拟（纸交易 3 个月后才考虑真实资金）
12. 日频 vs 周度策略 A/B 对比，选择一个统一执行

## 五、总体结论

| 维度 | 现状 | 能否辅助交易 |
|------|------|:--:|
| 特征工程 | 方向正确，覆盖较全 | 🟡 有用但不充分 |
| 回测验证 | 方法论正确，但没有跑出可展示的结果 | 🔴 缺少证据 |
| 实际跟踪 | 仅 1 条记录，0 条验证 | 🔴 完全缺失 |
| 执行可行性 | 日频 + T+1 错配，交易成本高 | 🔴 不可执行 |
| 风控体系 | 仅有简单止损止盈 | 🔴 严重不足 |
| A 股适配 | 未处理涨跌停、T+1、除权 | 🔴 致命 |
| 数据时效 | 离线 CSV，靠手动运行 | 🔴 无法实盘 |
| 过拟合控制 | 100+ 特征 / ~190 有效样本 | 🔴 高风险 |

**当前状态：** 这是一个「特征工程 + 模型训练」的练习级项目，代码框架花了心思，但尚未跨过从「模型能跑」到「模型能辅助决策」的门槛。

**核心问题不是模型不够复杂——而是没有证明它比扔硬币更好，也没有在任何真实场景中被验证过。** 在完成 Phase 1 之前，这个系统不应被用于任何真实资金决策。
